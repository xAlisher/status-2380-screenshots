Wallet

20 issues · Send flow, assets, token picker, account management Desktop 2026-05-18

Ways to Buy — single provider shown as full-screen modal

Issue#20903
FindingWhen only one provider is available, tapping "Ways to buy" opens a full-screen modal with a single list item. The extra navigation step adds no value — the single provider should open directly.
Impact✗ Unnecessary modal layer for one-option scenario
SuggestionIf exactly one provider is available, skip the picker and open the provider directly. Show the modal only when ≥2 providers are available.

Send — max button disabled with no explanation

Issue#20904
FindingThe "Max" button in the send flow is visually disabled (greyed out) with no tooltip or explanation. The fee row shows dashes instead of values. Users cannot tell if this is a loading state, insufficient balance, or a network error.
Impact✗ Blocked send action with no recovery guidance
SuggestionShow a loading spinner while fees are fetching. If fees fail to load, show an inline error with a Retry button. Tooltip on disabled Max: "Fees not yet loaded" or "Insufficient balance for fees".

Send — amount field visually clipped

Issue#20906
FindingThe amount input field clips the typed value — the leading digit is hidden by the left edge of the input container. The full amount is present but not visible without scrolling the field.
Impact✗ User cannot verify the full amount they are sending
SuggestionIncrease horizontal padding in the amount field, or shrink the font size when the entered value exceeds the visible width (similar to iOS calculator auto-shrink pattern).

Sign & Send — title block duplicates header, pushes details below fold

Issue#20907
FindingThe "Sign & Send" confirmation modal shows a large title block ("Sign & Send" heading + network/account row) that repeats the same info as the modal header, consuming ~40% of visible screen height and pushing the fee/amount details below the fold.
Impact✗ Transaction amount not visible without scrolling on confirmation screen
SuggestionCollapse the redundant title block. Show network + account inline with the header. Prioritise amount and fee breakdown above the fold.

Custom fees modal — clipped content, no scroll hint, no close

Issue#20908
FindingThe "Edit fees" modal clips its bottom content (max fee field cut off), shows no scroll indicator, and has no visible close/dismiss button. Users can get stuck in this modal on smaller screens.
Impact✗ User cannot see or set max fee; may be trapped in modal
SuggestionAdd explicit close (×) button. Add scroll-hint gradient at bottom of clipped content. Ensure modal height is bounded so all fields are reachable.

Token picker — search state resets on selection

Issue#20910
FindingAfter searching and selecting a token, reopening the token picker clears the search field and resets to the full list. The previously selected token is not highlighted or shown at top. Users who re-open the picker lose their search context.
Impact⚠ Repeated search friction when switching between tokens
SuggestionPin the currently-selected token at the top of the list. Persist the search query for the session or reset cleanly with the selection highlighted.

Collectible send — opens blank modal

Issue#20918
FindingTapping "Send" from a collectible detail page opens a blank send modal — no collectible name, image, or pre-filled recipient. The modal appears completely empty.
Impact✗ Cannot send collectibles from detail view; send flow broken
SuggestionPre-populate the send modal with the collectible context (image thumbnail, name, token ID, chain). Treat as a regression test case for collectible send.

Wallet tab indicator too close to content

Issue#20905
FindingThe active tab underline indicator on the wallet (Assets / Collectibles / History) sits flush against the first row of list content with no gap. On both tabs, the indicator visually bleeds into the content area.
Impact⚠ Minor visual polish issue, affects perceived quality
SuggestionAdd 4–8dp bottom padding below the tab bar before the content list begins.

Chat & Messaging

18 issues · Chat input, pinned messages, search, contact flow, activity pane Desktop 2026-05-12 & 2026-05-18

Send and scroll-to-bottom icons are identical

FindingThe "Send message" button and "Scroll to latest" floating button use visually identical chevron-up icons. Users tapping the wrong button may send an incomplete message or miss new messages.
Impact✗ Critical ambiguity — destructive if send is triggered unintentionally
SuggestionUse a paper-plane or arrow-right icon for Send. Reserve chevron-up/down exclusively for scroll actions.

Context menu missing Copy action

FindingLong-pressing a chat message shows a context menu without a "Copy" option. Copy is a fundamental expectation in all chat products; its absence forces users to select text manually.
Impact✗ Core interaction missing; high user-reported frustration expected
SuggestionAdd "Copy text" as the first item in the message context menu.

Pinned message banner is not interactive

FindingThe pinned-message banner at the top of a channel is shown but tapping it does nothing — it does not scroll to the pinned message or open the pinned messages panel. The banner sets an expectation it does not fulfill.
Impact✗ Dead interactive element; misleads user expectation
SuggestionTapping the banner should scroll the view to the pinned message. A secondary tap or swipe could open the full pinned messages list.

Activity pane expands the left column instead of overlaying

FindingOpening the notification/activity pane pushes the left navigation column wider instead of overlaying. This compresses the channel list and chat view simultaneously, causing layout reflow visible to the user.
Impact⚠ Disorienting layout shift; reduces screen real estate for primary content
SuggestionRender the activity pane as a floating overlay anchored to the right, not as a flexbox sibling of the nav rail.

Cross-chat search results have no navigation

FindingThe cross-chat search returns results from multiple channels, but tapping a result does not navigate to the message in context. Results are listed without a way to jump to the message.
Impact✗ Search results are decorative — core navigation intent is blocked
SuggestionTapping a search result should navigate to the channel and scroll-highlight the matching message.

Contact list not fetched — empty state with no guidance

FindingThe contacts list shows empty with no explanation of whether it is loading, failed, or genuinely empty. No "Add contact" prompt or search suggestion is shown to the new user.
Impact✗ Dead end for new users; they have no path to finding or adding contacts
SuggestionShow a distinct loading state while fetching. If empty, show an actionable empty state: "No contacts yet — share your profile link or search by ENS / Status ID."

Share profile icon opens preview instead of QR code

FindingTapping the share/QR icon on a contact's profile opens a web-style profile preview page rather than a QR code for scanning. The icon communicates QR scanning; the result is a different action.
Impact⚠ Icon-action mismatch; adds friction for in-person contact sharing
SuggestionUse a distinct icon for "View profile" vs "Show QR code". Default the share icon to QR display; offer a secondary link to the profile preview.

Communities

6 issues · Share URL, discovery, empty state, Android-specific Desktop Android 2026-05-18

Share dialog shows user profile URL instead of community URL

Issue#20923
FindingOpening the Share dialog from a community page shows the current user's profile URL, not the community's URL. Anyone who shares this link sends their own profile to others instead of the community.
Impact✗ Sharing is broken — community growth path uses wrong link
SuggestionWhen the share dialog is opened from a community context (community header, community settings), populate the link with status.app/c/{communityKey} not status.app/u/{profileKey}.

Communities directory shows nothing on first open — no loading state

Issue#20924 Android
FindingOn first open, the Communities directory is completely empty with no loading indicator, no error message, and no explanation that communities are being fetched from the network. A new user who sees this is likely to assume the feature is broken and not return.
Impact✗ Immediate churn risk for new users exploring communities
SuggestionMinimum: show a "Fetching communities…" message with a spinner. Better: embed a small set of curated/featured communities in the build so the screen is never empty on first launch.

Discover Communities search returns no results for name queries

Issue#20926 Android
FindingTyping any community name in Discover Communities always shows "No communities found." The search fires on text change against a local cache that is empty on first launch. There is no submit button, no debounce-then-network-search, and no indication that the search requires the cache to be populated first.
Impact✗ Community discovery by name is non-functional; only community key paste works
SuggestionFall back to a network search when the local cache returns no results. Add a Submit/Search button. Show hint text: "Search works after communities load — paste a community link to join immediately."

App crashes when tapping community name header in guest mode

Issue#20925 Android
FindingTapping the community name/header banner in guest mode (not a member) crashes the app to the Android home screen. Reproducible on second tap after cold launch (confirmed 2×). No crash dialog or error shown.
Impact✗ Hard crash; guest community browsing is unreliable
SuggestionAdd a null-check or guard in the community header tap handler for the case where the user is not a member. File for crash symbolication — this is a signal of a missing membership state guard in the navigation stack.

Profile & Settings

5 issues · QR modal, copy link, profile sharing Desktop 2026-05-12

Profile QR modal content cut at bottom

FindingThe profile QR code modal clips its bottom content — the share link and copy-link buttons are partially visible or fully cut off without a scroll affordance. On some window sizes the QR code itself is partially obscured.
Impact✗ Copy-link CTA inaccessible; profile sharing is broken at small window heights
SuggestionMake the modal scrollable, or constrain the QR code size so that all action buttons are always visible above the fold. Add a minimum-height media query or dynamic sizing.

Copy profile link shows no feedback

FindingTapping "Copy link" on the profile page copies to clipboard but shows no visual confirmation — no toast, no icon change, no checkmark. Users cannot tell if the action succeeded.
Impact⚠ No confirmation of clipboard write; users may tap multiple times
SuggestionShow a brief "Copied!" toast or animate the copy icon to a checkmark for 1.5s after the clipboard write succeeds.

Profile QR avatar has no border or branding

FindingThe profile QR code shows a raw QR matrix with no Status logo center-inset, no border/padding around the code, and no context label. Compared to competitor wallets (MetaMask, Rainbow), this is the least polished QR presentation.
Impact⚠ Perceived quality gap; missed branding opportunity on a highly-shared screen
SuggestionAdd the Status logo (or user avatar) center-inset into the QR code. Add 12dp white padding around the QR matrix. Add "Scan to add me on Status" label below.
